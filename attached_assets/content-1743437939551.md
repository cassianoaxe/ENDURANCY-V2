# Integrations Toolkit Azul Cargo (v1)

# [section/Introduction](https://hmg.onlineapp.com.br/EDIv2_API_INTEGRACAO_Toolkit/Home/English\#section/Introduction) Introduction

The Azul Cargo Integration Toolkit facilitates the integration of Azul Cargo clients, providing services of Pricing, Tracking, and Issuance of CTes (AWBs) in an automatic and simple way, through APIs, where the client will be able to perform their own implementation, and thus be connected to the cargo platform of Azul Brazilian Airlines.

# [section/Return-of-codes](https://hmg.onlineapp.com.br/EDIv2_API_INTEGRACAO_Toolkit/Home/English\#section/Return-of-codes) Return of codes

- **200 Ok:** Occurs when the request is successfully completed and the expected return is obtained.


- **201 No content:** Occurs when the request is successfully completed, but no content is found to return according to the parameters provided for the request.


# [section/Error-codes](https://hmg.onlineapp.com.br/EDIv2_API_INTEGRACAO_Toolkit/Home/English\#section/Error-codes) Error codes

- **400 Invalid Request:** Occurs when the request has invalid arguments to complete the request.


- **401 Unauthorized:** Occurs when the user is not properly authenticated (Invalid Token) or when the user does not have permission to access the requested resource.


- **403 Forbidden:** Occurs when the server receives the request and was able to identify the author, but did not authorize the issuance of a response.


- **404 Not Found:** Occurs when the call does not find the destination path of the request, either because of the wrong destination URL or because the requested service is not available.


- **500 Internal Server Error:** Occurs when the server is unable to successfully complete the requested request.

If you have another commercial agreement negotiated in another service, different from the standard services (EXPRESSO, STANDARD), please contact the Azul Cargo Express business services.

## [operation/FaturasOnline](https://hmg.onlineapp.com.br/EDIv2_API_INTEGRACAO_Toolkit/Home/English\#operation/FaturasOnline) Online Invoice

Returns the list of invoices for a given CNPJ within the informed query period.

Sending the CNPJ field is optional and used if the authenticated user has access to more than one company.

By default, the system will consider the user's company(ies) linked to the token provided for query.

If a CNPJ is informed to which the user does not have access permission, the request will return with code 403.

The approval/production access credentials (username and password) are sent in the email used to register the toolkit.

You can also import the examples of requests from this documentation in your Postman. For this, just use [this link](https://api.postman.com/collections/37174765-7476afb4-ca1a-4004-9b5e-307d7797d878?access_key=PMAT-01J3FSJK40FX25KP94J3FR660A) and import from collection.

**Warning:** All fields marked as **Required** are **mandatory**

##### Request Body schema: application/json

|     |     |
| --- | --- |
| Token<br>required | string<br>Security key after system authentication. |
